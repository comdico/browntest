What is semantic HTML?

Semantic HTML defines the structure of web documents so that it's markup adds contextual meaning to it that can improve comprehansion by humans as well as machines. Semantic markup can both help structure a document at the macro level by using the HTML outline model to create the correct document structure, and through the use of specific tags (HTML and micro-formats) that help define a dopcument's meaning at the micro level.

What is accessibility?

HTML documents should be accessible be as many devices as possible, including screen readers. Many people in the world still access web content via older browsers or cell phones with older embedded browsers.

Accessibility for screen readers is best acheived by incorporating WAI-ARIA attributes, which help define a page's structure for specialized web-clients for people with disabilities.

The WAI-ARIA spec is now a formal part of the W3C specification.

What is progressive enhancement?

Progressive enhancement is a strategy for building webpages that are semantic, accessible, and work in as many browsers as possible.

As oppossed to graceful degradation, which treats accessibility from the standpoint of a finished site with all the bells and whistles, progressive enhancement, builds the core product with the most basic of tools, and then adds enhancements, such as CSS3 and Javascript to layer in complex functionality

Responsive design and content-out is an out growth of progressive enhancement.
